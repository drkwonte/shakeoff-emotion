# ShakeOff PRD

## 1. 제품 개요

**제품명**: ShakeOff  
**슬로건**: 힘든 감정을 불태우고 후련하게 털어내는 디지털 감정 소각로

**목적**:  
사용자가 부정적인 감정을 자유롭게 쏟아낸 후, 강렬한 “불태우기” 의식을 통해 catharsis(감정 정화)를 경험하고, 시각적으로 밝아지는 해방감을 느끼게 한다. 기록, 저장, 공유, 통계 등 어떤 형태의 데이터도 남기지 않는다.

**핵심 가치**:

- 완전한 일회성 (한 번 불태우면 절대 복구 불가)
- 강렬한 catharsis + 후련함
- “기억하지 않는다”는 철학

## 2. 타겟 사용자

- 감정을 쌓아두고 스트레스 받는 20~35세
- 일기나 기록 앱은 부담스러운 사람
- 강렬한 감정 배출을 원하는 사람
- 프라이버시를 매우 중요하게 생각하는 사용자

## 3. 주요 기능 및 사용자 흐름

### 3.1 입장 화면 (Landing)

- 메인 문구: “힘든 일이 있나요? (줄바꿈) 저에게 털어놓으세요.”
- 보조 문구: “판단도, 조언도, 기록도 없습니다. 마음껏 쏟아내고 불태우세요.”
- “시작하기” 버튼

### 3.2 감정 작성 화면

- 넓은 textarea (자유롭게 작성 가능)
- 감정 태그 버튼 (선택사항): 분노, 우울, 불안, 자책, 억울함, 짜증, 외로움 등
- 하단에 큰 버튼: **“불태우기”**
  - 작성량이 일정 수준 이상이 되어야 버튼이 활성화 (붉은색으로 변경)

### 3.3 불태우기 의식 (Core Experience)

- “불태우기” 버튼 클릭 시 확인 모달 표시:
  - “정말 모든 것을 불태우시겠습니까? 한 번 불태우면 절대 되돌릴 수 없습니다.”
- 확인 후:
  - 강렬한 불꽃 + 연기 + 타는 종이 파티클 애니메이션 (Canvas + TS Particles)
  - 작성한 모든 텍스트가 실제로 불타면서 사라짐
  - 모든 것이 타서 **재가 된 후 바람에 날아가는** 연출
  - 재가 거의 사라진 후, 배경이 서서히 **어두운 톤 → 밝고 따뜻한 sunrise 톤**으로 전환 (주황 → 코랄 → 밝은 민트/베이지)

### 3.4 완료 화면

- 중앙에 메인 문구: **“고생하셨어요. 이제 후련하시죠?”**
- 밝고 상쾌한 배경 유지
- 버튼: **“하나 더 있는데…”** (클릭 시 처음 작성 화면으로 초기화)

## 4. 중요 제약사항 (Non-Negotiable)

- **어떠한 형태로도 데이터 저장 금지**
  - localStorage, sessionStorage, IndexedDB, Cookie 모두 사용 금지
  - 페이지 새로고침, 뒤로가기, 브라우저 종료 시 모든 데이터 완전 소멸
- 공유 기능, export 기능, 기록 보기 기능 전부 없음
- undo, 되돌리기 기능 절대 금지

## 5. 디자인 방향

- 초기: Dark & Intense (Deep Charcoal, Crimson accent)
- 불태우는 순간: 강렬한 Fire Particle Effect
- 완료 후: Bright & Refreshing (Warm Sunrise → Soft Mint & Beige)
- 전체적으로 의식(ritual) 같은 몰입감 있는 UI/UX

## 6. 기술 스택

- **프론트엔드**: React + TypeScript + Vite
- **스타일링**: Tailwind CSS 또는 Styled-Components
- **애니메이션**:
  - Canvas (Particles.js 또는 tsParticles)
  - Framer Motion (UI 전환)
  - GSAP (고급 파티클 및 전환 효과)
- **배포**: Cloudflare Pages
- **기타**: 완전 Client-side only (백엔드 없음)

## 7. MVP 기능 범위 (Must-have)

1. 감정 작성 화면
2. “불태우기” 버튼 + 확인 모달
3. 강렬한 불태우기 애니메이션 (불 → 재 → 날아감)
4. 배경 밝아지는 전환 효과
5. 완료 화면 (“고생하셨어요. 이제 후련하시죠?” + “하나 더 있는데…” 버튼)
6. 데이터 저장 완전 차단 (모든 상태는 메모리에만 존재)

## 8. Nice-to-have (추후 고려)

- 음성 입력 (Speech-to-Text)
- 더 정교한 파티클 효과
- 모바일 최적화 강화
- 다크/라이트 모드 전환 (완료 후)
